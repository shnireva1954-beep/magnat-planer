# Магнат — память проекта

Продукт: набор из 5 Google-таблиц с игровой механикой тайкуна (990 ₽ разово)
+ бонус: веб-приложение `app.html`. Владелец — самозанятый Могуленко А.Е.
Сайт: https://magnat-planer.ru (GitHub Pages из ветки `main`, корень репо).

## Архитектура
- Статика, без сборки: `index.html` (лендинг), `app.html` (приложение,
  localStorage `magnat_app_v2`), `oferta.html`, `privacy.html`,
  `manifest.json` + `icon.png`/`icon-512.png` (PWA), `og.png` (1200×630).
- Весь CSS/JS инлайн в HTML. Никаких зависимостей.
- Рабочая ветка: `claude/website-audit-conversion-g8gx9n`.
- **Публикация** = fast-forward `main` до ветки + push (только по команде
  владельца «публикуй»). После публикации OG-кэш Telegram сбрасывается
  отправкой URL боту @WebpageBot.

## Что уже сделано (июль 2026, большая сессия аудита)
1. Аудит + доверие: trust-бейджи у цены, ценовое якорение (990 разово vs
   ~3600 ₽/год подписок), versus-блок, якорная навигация, FAQ (7 пунктов,
   первый — «Почему таблицы, а не приложение?»), FAQPage JSON-LD.
2. Превью планеров = премиум-дашборды в рамке «настоящего Google Sheets»:
   KPI-плитки с дельтами, инсайт-строки 💡, радар мышц (Фитнес), кривая
   энергии (Неделя), копилка по месяцам (Финансы), вкладки листов снизу.
   Вся арифметика в превью сведена (категории = итогам и т.п.).
3. Эмоция + сдержанность (правило: **одна яркая эмоция = одно достижение**):
   демо-игра в hero (монеты летят в кошелёк, империя 🛖→👑 строится, салют
   ТОЛЬКО за идеальную неделю/Магната), в приложении празднуется только
   новый уровень. Конфетти на каждом действии — нельзя.
4. Приложение: празднование уровня, полёт монет, честный «Инсайт недели»
   (только из реальных данных; НИКОГДА не выдумывать статистику типа «лучше
   84% пользователей»), экран «Тело» с путём к цели и редактируемой целью,
   календарь привычек (sticky-имена, автоскролл к сегодня), редактирование
   задач, свои награды, экспорт/импорт данных (бэкап JSON), PWA-иконка,
   демо-фрейминг (плашка «бесплатное демо → 990 ₽», баннер демо-данных).
5. Финальный техпасс: doctype+lang=ru на всех страницах (был quirks mode!),
   типографские тире вместо дефисов, контраст, tap-targets, noindex на
   app.html, OG-метаданные, «рекорд серии» вместо фейковой «заморозки».

## Принципы (не нарушать)
- **Честность**: никаких выдуманных отзывов, фейковых счётчиков покупателей,
  ложных гарантий (по оферте п.5.2 возврата за цифровой товар нет — бейдж
  «гарантия возврата» ставить нельзя). Всё в превью планеров должно быть
  реализуемо штатными средствами Google Sheets.
- Сдержанность эффектов; prefers-reduced-motion уважается везде.
- Тексты — с настоящими тире «—», не дефисами.

## ОЖИДАЕТ ВЛАДЕЛЬЦА (следующие шаги, по приоритету)
1. **Реальные Google-таблицы** по ТЗ `SHEETS-SPEC.md` (в корне репо).
   До их соответствия превью продавать нельзя. После готовности: заменить
   превью реальными скриншотами + дать ссылку «Открыть демо-таблицу».
2. **Робокасса**: владелец пришлёт платёжную ссылку → вставить в
   `PAYMENT_URL` (index.html, искать `var PAYMENT_URL`); кнопки «Купить»
   заработают, модалка-бронь отключится сама. Письмо после оплаты со
   ссылкой на таблицы настраивается в ЛК Робокассы (п.4.1 оферты).
3. **Почта на домене** (сейчас shnireva1954@gmail.com при продавце
   Могуленко — дыра доверия): заменить в index.html (футер, модалка,
   mailto в JS), oferta.html (3 места), privacy.html (2 места).
4. **Отзывы**: в index.html есть закомментированный блок-шаблон (искать
   «БЛОК ОТЗЫВОВ») — заполнить ТОЛЬКО реальными и раскомментировать.
5. **Яндекс.Метрика**: владелец пришлёт номер счётчика → вставить код.
6. Telegram-бот помечен «скоро» (3 места в index.html) — сервера нет.

## Как верифицировать изменения
Chromium: `/opt/pw-browsers/chromium-1194/chrome-linux/chrome` через
`playwright-core` (ставить `npm i -D playwright-core`, НЕ коммитить
node_modules/package*.json — удалять перед коммитом). Проверять: десктоп
1280 + мобайл 390 (h-scroll, ошибки консоли), демо-дугу (15 кликов →
ровно 1 салют → Магнат), сценарии приложения на сид-данных и fresh-start.
Превью планеров скейлятся scale-to-fit (fit() в index.html) — на мобиле
KPI перестраиваются в 2 колонки, чтобы лист был крупнее.
