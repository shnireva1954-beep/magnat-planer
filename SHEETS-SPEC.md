# ТЗ: реальные Google-таблицы «Магнат 5 в 1» — v2

**Цель:** каждый реальный лист визуально и функционально соответствует превью на
magnat-planer.ru. Всё в этом ТЗ реализуемо штатными средствами Google Sheets —
превью проектировались с этим ограничением. Критерий готовности: скриншот
реального дашборда не стыдно поставить на лендинг вместо макета.

**Продавать до соответствия таблиц превью нельзя** — это прямой путь к
возвратам («на сайте не то, что мне прислали»).

**Главный принцип v2 — ни одного пустого белого пятна.** Каждый блок дашборда
либо показывает живую аналитику по данным пользователя, либо его нет. При этом
не перегружать: на дашборде максимум 6–7 блоков, один инсайт, одна эмоция.

---

## 0. Общая структура каждого файла

Каждый планер — отдельный файл Google Sheets. Состав листов (имена вкладок —
ровно как в превью на лендинге):

| Лист | Назначение |
|---|---|
| **📊 Дашборд** | Витрина: KPI-плитки, диаграммы, инсайт. Пользователь сюда смотрит |
| **Рабочие листы** (Привычки/Задачи/…) | Сюда ставят галочки и вводят записи |
| **Настройки** | Привычки и цены в монетах, цели, пороги уровней + инструкция |
| *(скрытый)* **_calc** | Промежуточные формулы, чтобы Дашборд был чистым |

Вкладки по планерам (сверено с превью):
- Привычки: `📊 Дашборд · Привычки · История · Настройки`
- Задачи: `📊 Дашборд · Задачи · Архив` (+ Настройки можно спрятать в Архив)
- Тело: `📊 Дашборд · Тренировки · Замеры` (+ Настройки)
- Неделя: `📊 Дашборд · Неделя · Рефлексия` (+ Настройки)
- Финансы: `📊 Дашборд · Операции · Долги · Цели` (+ Настройки)

**Оформление** в стиле превью: белый фон, шапки `#eef2f8`, сетка `#e0e6ef`,
акценты: изумруд `#12a565`, золото `#c98a12`, красный `#e0556a`,
фиолетовый `#7a52d0`, синий `#2f8fd8`. KPI-плитки — объединённые ячейки с
белой заливкой, тонкой границей и цветной левой «полосой» (узкий столбец
с заливкой акцентного цвета). Первая строка/столбец дашборда — узкие поля,
чтобы блоки «дышали».

**Защита (обязательно):** всё, кроме ячеек ввода, защитить
(Данные → Настроить защищённые листы → «Кроме отдельных ячеек»).
Иначе пользователи стирают формулы — главная причина обращений в поддержку.
Лист `_calc` — скрыть и защитить целиком, в A1 предупреждение:
«⚙️ Служебные формулы. Не редактируй — сломается дашборд».

## 1. Игровая механика (одинаково во всех планерах, сверено с приложением)

- **Монеты за день** = `СУММПРОИЗВ(галочки_дня; цены_привычек)`
  (цены задаются в Настройках, по умолчанию 10–30).
- **День засчитан** в серию, если выполнено ≥ 3 привычек:
  `=ЕСЛИ(СЧЁТЕСЛИ(диапазон_дня;ИСТИНА)>=3;1;0)`.
- **Серия** — количество зачтённых дней подряд до сегодня (в _calc:
  бегущий счётчик `=ЕСЛИ(день_зачтён=1;пред_серия+1;0)`).
- **Множитель** = `МИН(2; 1 + 0,1 * серия_до_вчера)`.
- **Монеты с множителем** = `ОКРУГЛ(монеты_дня * множитель)`.
- **Уровень** по суммарным монетам, пороги как в приложении:
  `=ПРОСМОТР(всего_монет; {0;300;1500;5000;15000}; {"🐣 Новичок";"🚀 Стартапер";"💼 Предприниматель";"🏢 Директор";"👑 Магнат"})`.
- Прогресс до следующего уровня — SPARKLINE-бар:
  `=SPARKLINE(монет_в_уровне; {"charttype";"bar"; "max";размер_уровня; "color1";"#12a565"})`.

**Анти-ошибки (правило для всей механики):** каждая формула дашборда обёрнута
в `ЕСЛИОШИБКА(…; фолбэк)`, деления защищены `ЕСЛИ(знаменатель=0; 0; …)`.
Свежая копия без единой галочки обязана показывать: серия 0 · множитель ×1,0 ·
монеты 0 · уровень 🐣 Новичок — и ни одной ошибки в ячейках.

## 2. Планер привычек — «Штаб дисциплины»

**Лист Привычки:** строки — привычки (эмодзи+название подтягиваются из
Настроек через `=Настройки!A2` и т.д.), столбцы — дни месяца, ячейки —
чекбоксы (Вставка → Флажок). Колонка `%` =
`=ЕСЛИОШИБКА(СЧЁТЕСЛИ(строка;ИСТИНА)/МАКС(1;прошедшие_дни);0)`.
Лист История: по строке на месяц — итоги (монеты, % месяца, лучшая серия),
переносятся вручную или формулой в конце месяца.

**Дашборд (сверху вниз, как в превью):**
1. **KPI-плитки ×4**: Серия 🔥 · Множитель · Монеты сегодня · Уровень
   (формулы из п.1).
2. **Кривая прогресса** — диаграмма «Область» по % выполнения каждого
   прошедшего дня (ряд в _calc: `=ЕСЛИ(дата>СЕГОДНЯ();"";% дня)` — будущие
   дни пустые, линия не падает в ноль). Высота — как у пончика рядом, без
   пустых полос.
3. **Пончик «Месяц»** — круговая с отверстием: выполнено/осталось галочек
   месяца. Процент в центре: поверх отверстия ставится **сводная карточка**
   (Scorecard) с `=ЕСЛИОШИБКА(галочки/(привычки*прошедшие_дни);0)` в формате
   `0%`, фон карточки — «Нет» (прозрачный). Два плавающих объекта, карточка
   поверх — выглядит ровно как в превью.
4. **Тепловая карта месяца** — НОВОЕ, главный «вау»-блок. Диапазон-календарь
   7 столбцов (пн–вс) × 5–6 строк, в каждой ячейке % дня из _calc:
   `=ЕСЛИ(дата_ячейки>СЕГОДНЯ();"";% дня)`.
   Условное форматирование, правила по порядку:
   - «Ячейка пуста» → без заливки (будущие дни остаются белыми);
   - пользовательская формула `=дата_ячейки=СЕГОДНЯ()` → светло-золотая
     заливка `#fdf1d7` (метка «сегодня»; рамку УФ не умеет — только заливку);
   - «Шкала цвета»: минимум `#eef2f7` (пропуск) → максимум `#12a565`
     (идеальный день).
   Число в ячейке скрыть цветом текста = цвету заливки или форматом `;;;`.
   Подписи пн–вс над картой — обычная строка мелким серым капсом.
5. **Таблица недели**: срез листа Привычки за текущую неделю
   (`=ФИЛЬТР(…; даты>=начало_недели; даты<=СЕГОДНЯ()+…)`), с колонкой %.
6. **Прогресс уровня** — SPARKLINE-бар (п.1) + подпись
   `="осталось "&ТЕКСТ(порог-монеты;"# ##0")&" 🪙 до "&след_уровень`.
7. **Инсайт** (одна ячейка, стиль «💡 на светло-жёлтом»):
   `=ЕСЛИОШИБКА("Твой сильный день — "&ИНДЕКС(дни;ПОИСКПОЗ(МАКС(проценты_по_дням);проценты_по_дням;0))&" ("&ТЕКСТ(МАКС(проценты_по_дням);"0%")&"). Ещё 1 день серии — множитель "&ТЕКСТ(МИН(2;1,1+0,1*серия);"0,0")&".";"Отмечай привычки — здесь появятся инсайты")`.

## 3. Планер задач — «Центр задач»

**Лист Задачи:** колонки Задача · Срок (дата) · Приоритет (раскрывающийся
список Срочно/Высокий/Средний/Низкий) · ✓ (чекбокс) · Дата выполнения
(проставляется при закрытии; можно вручную, колонку можно скрыть).
Колонка «Осталось» = `=ЕСЛИ(✓;"✓";ЕСЛИ(срок="";"—";срок-СЕГОДНЯ()))`.
Лист Архив: выполненные строки переносятся (Ctrl+X/Ctrl+V) раз в неделю —
рабочий список остаётся коротким.

**Условное форматирование:** просрочка (`=И(осталось<0; НЕ(✓))`) — красный
текст; выполненные — серые + зачёркнутые (формат по ✓=ИСТИНА); приоритеты —
цветные пилюли (заливка по значению: Срочно/Высокий `#fbe9ec`, Средний
`#fdf1d7`, Низкий `#e3f0fb`).

**Дашборд:**
1. **KPI ×4**: Открыто (`СЧЁТЕСЛИ(✓;ЛОЖЬ)`) · Горит
   (`СЧЁТЕСЛИМН(срок;"<"&СЕГОДНЯ();✓;ЛОЖЬ)`) · Закрыто за неделю
   (`СЧЁТЕСЛИМН(дата_вып;">="&СЕГОДНЯ()-6)`) · % в срок
   (`=ЕСЛИОШИБКА(в_срок/МАКС(1;закрыто);0)` форматом `0%`).
2. Таблица ближайших задач: `=СОРТИР(ФИЛЬТР(…;✓=ЛОЖЬ);столбец_срок;ИСТИНА)`
   — топ-5 по сроку.
3. **Столбчатая «Закрыто по дням»** — НОВОЕ рядом с пончиком, в две колонки.
   Ряд в _calc: `=СЧЁТЕСЛИМН(дата_вып;ячейка_дня)` для пн–вс текущей недели.
   Заголовок ряда: «Закрыто по дням · N за неделю» (N из KPI).
4. **Пончик «Приоритеты»** — НОВОЕ: круговая с отверстием по открытым
   задачам. Ряды в _calc: Срочно+Высокий / Средний / Низкий =
   `=СЧЁТЕСЛИМН(приоритет;"Срочно";✓;ЛОЖЬ)+СЧЁТЕСЛИМН(приоритет;"Высокий";✓;ЛОЖЬ)`
   и т.п. Цвета сегментов: `#e0556a` / `#c98a12` / `#2f8fd8`. Число «открыто»
   в центре — прозрачная Scorecard поверх отверстия (как в п.2.3).
5. **Инсайт**: `=ЕСЛИОШИБКА("Пик продуктивности — "&день_МАКС_закрытий&"."&ЕСЛИ(горит>0;" «"&ИНДЕКС(задачи;ПОИСКПОЗ(МИН(сроки_просроченных);сроки;0))&"» просрочена — закрой её первой.";" Просрочек нет — так держать!");"Добавь задачи со сроками — аналитика появится сама")`.

## 4. Фитнес-планер — «Панель формы»

**Лист Тренировки (дневник):** Дата · Вес · Ккал · Вода · Сон · Тренировка
(чекбокс) · Группа мышц (список) · Настроение. **Лист Замеры:** дата · талия ·
грудь · бицепс и т.п. **Настройки:** стартовый вес, цель веса, норма
ккал/воды, план тренировок в неделю.

**Дашборд:**
1. **KPI ×4**: Вес (последний: `=ЕСЛИОШИБКА(ПРОСМОТР(2;1/(веса<>"");веса);"—")`)
   · Тренировки за неделю `X / 7` (+ подпись «план N» из Настроек) · Ккал
   (последние: та же ПРОСМОТР-формула, подпись «цель N») · Сон средний за
   7 дней (`=ЕСЛИОШИБКА(СРЗНАЧЕСЛИМН(сон;дата;">="&СЕГОДНЯ()-6);"—")`).
2. **График веса** — «Область»: серия веса + константная серия «цель»
   (колонка в _calc, залитая значением цели из Настроек) → **пунктирная
   золотая линия цели** прямо на графике (стиль линии серии: пунктир,
   `#c98a12`, подпись серии «цель N»). Пустые дни ряд пропускает:
   `=ЕСЛИ(вес="";"";вес)` + «Показывать пустые ячейки: пропуск».
3. **Радар «Баланс мышц»** — диаграмма «Лепестковая» по `СЧЁТЕСЛИМН`
   подходов на каждую группу за 30 дней. Radar есть в Sheets: тип
   «Лепестковая».
4. **Путь к цели** — НОВОЕ: строка «🎯 Путь к цели» с SPARKLINE-баром
   `=SPARKLINE(МАКС(0;МИН(1;ЕСЛИОШИБКА((старт-тек)/(старт-цель);0)));{"charttype";"bar";"max";1;"color1";"#c98a12"})`
   и подписью `=старт&" → "&тек&" → "&цель&" кг · пройдено "&ТЕКСТ(доля;"0%")`.
5. **Статистика 30 дней** — НОВОЕ, три плитки-«стата»:
   Тренировок за 30 дн (`СЧЁТЕСЛИМН(трен;ИСТИНА;дата;">="&СЕГОДНЯ()-29)`) ·
   Лучшая серия тренировок (бегущий счётчик в _calc, `МАКС`) ·
   Средний вес недели (`СРЗНАЧЕСЛИМН`).
6. **Тренировки недели** — строка-пилюли пн–вс: `=ЕСЛИ(была_тренировка;
   день&" ✓";день)`, УФ: содержит «✓» → зелёная заливка.
7. **Инсайт**: `=ЕСЛИОШИБКА(МИН_группа_текст&" отстаёт — добавь один подход, и радар выровняется.";"Заполни дневник — инсайты появятся здесь")`.

## 5. Планер недели — «Пульт недели»

**Лист Неделя:** 7 блоков дня: 3 главных дела (чекбоксы) · энергия (1–5) ·
настроение (список эмодзи). Отдельно: 3 фокус-цели недели (чекбоксы).
**Лист Рефлексия:** 3 вопроса недели (что получилось / что мешало / что
изменю) — по строке на неделю.

**Дашборд:**
1. **KPI ×4** (в превью теперь четыре): Выполнено X/Y · Энергия средняя
   (`=ЕСЛИОШИБКА(СРЗНАЧ(энергия);"—")`, подпись «пик — …» через
   ИНДЕКС/ПОИСКПОЗ по МАКС) · Фокус-цели X/3 · **Монеты за неделю** —
   НОВОЕ: `=СЧЁТЕСЛИ(дела;ИСТИНА)*цена_дела` (цена дела в Настройках,
   по умолчанию 20–25) — игровая механика видна и здесь.
2. **Мини-карточки дней** (как в превью): дела-полоски, шкала энергии
   `=SPARKLINE(энергия;{"charttype";"bar";"max";5})`, настроение-эмодзи,
   кольцо дня — мини-пончик или `ТЕКСТ(процент;"0%")` с цветом.
3. **Кривая энергии** — «Область» по энергии 7 дней, фиолетовая `#7a52d0`,
   **с подписями пн–вс по горизонтальной оси** (ось подписывается из
   колонки дней — пустых областей под графиком нет).
4. **Фокус-цели** — пилюли с ✓/○, УФ по чекбоксу.
5. **Инсайт**: `=ЕСЛИОШИБКА("Энергия проседает к "&день_МИН&". Ставь лёгкие задачи на "&день_МИН&" — процент недели вырастет.";"Отмечай дела и энергию — инсайт появится в конце дня")`.

## 6. Финансовый планер — «Капитал»

**Лист Операции (журнал):** Дата · Тип (доход/расход) · Категория (список
с эмодзи) · Сумма. **Лист Цели:** категория → план на месяц; цель «% дохода
в сбережения»; цель подушки в месяцах (по умолчанию 6). **Лист Долги:**
кому · сумма · дата возврата, прогресс-бар SPARKLINE на каждую строку.

**Дашборд:**
1. **KPI ×4**: Доход · Расход (`СУММЕСЛИМН` за текущий месяц) · Сбережения
   (=Д−Р, подпись `=ТЕКСТ(ЕСЛИ(доход=0;0;сбер/доход);"0%")&" дохода"`) ·
   Подушка (`=ЕСЛИОШИБКА(накоплено/средний_расход_3мес;0)` в «X,X мес»,
   подпись «цель — 6»).
2. **Пончик расходов** по категориям (`СУММЕСЛИ` по категориям → круговая
   с отверстием). В центре — сумма расхода: прозрачная Scorecard поверх
   отверстия (п.2.3). **Легенда с процентами** — НОВОЕ: рядом/под пончиком,
   штатная легенда диаграммы (позиция «справа», метка «процент») или колонка
   `=ТЕКСТ(сумма_кат/расход;"0%")` в мини-таблице.
3. **План vs факт**: таблица категорий + SPARKLINE-бары факта
   (`{"charttype";"bar";"max";план}`), факт>план — УФ красным текстом.
4. **Копилка по месяцам** — столбчатая по `СУММЕСЛИМН` (доход−расход) за
   последние 6 месяцев (ряд в _calc через `ДАТАМЕС(нач_месяца;-5…0)`).
5. **Прогноз подушки** — НОВОЕ: строка «🛡️ Подушка X / 6 мес» с
   SPARKLINE-баром `=SPARKLINE(подушка_мес;{"charttype";"bar";"max";6;"color1";"#2f8fd8"})`
   и подписью `=ЕСЛИ(средние_сбер<=0;"начни откладывать — прогноз появится";"темп +"&ТЕКСТ(средние_сбер;"# ##0 ₽")&"/мес → цель через ~"&ОКРУГЛВВЕРХ((цель_мес*средний_расход-накоплено)/средние_сбер;0)&" мес")`.
   Честная арифметика, никаких выдуманных обещаний.
6. **Инсайт**: `="Сбережения "&ТЕКСТ(доля;"0%")&" дохода"&ЕСЛИ(есть_перерасход;" · "&категория&" превысила план на "&ТЕКСТ(дельта;"# ##0 ₽")&" — проверь её первой";" — выше цели, так держать")`,
   всё в `ЕСЛИОШИБКА(…;"Добавь операции — аналитика появится сама")`.

## 7. Аудит устойчивости (обязателен перед сдачей каждого планера)

**7.1. Пустая копия (главный тест).** Сделать «Файл → Создать копию», стереть
все демо-данные листов ввода (не формулы!) и проверить: ни одной ячейки с
`#REF!`, `#N/A`, `#VALUE!`, `#DIV/0!`; KPI показывают нули/тире; инсайт
показывает приветствие («Отмечай … — появятся инсайты»); диаграммы пустые,
но не сломанные.

**7.2. Формулы.**
- Только открытые диапазоны (`B2:B`, не `B2:B100`) — добавление строк
  пользователем ничего не ломает.
- Каждое деление — через `ЕСЛИ(знаменатель=0;…)` или `ЕСЛИОШИБКА`.
- `ПРОСМОТР(2;1/(…<>"");…)` для «последнего значения» — обёрнут в
  `ЕСЛИОШИБКА(…;"—")`.
- Ссылки между листами — на именованные диапазоны (Данные → Именованные
  диапазоны): переименование листа не даёт #REF!.
- Никаких волатильных массивов на тысячи строк (СЕГОДНЯ/ФИЛЬТР — ок;
  избегать ДВССЫЛ и сквозных СУММПРОИЗВ по всему листу) — таблица обязана
  открываться быстро и на телефоне.

**7.3. Диаграммы.** Диапазоны рядов — с запасом вниз (весь столбец _calc);
«Показывать пустые ячейки: пропуск»; цвета рядов — строго из палитры п.0;
у каждой диаграммы задан заголовок мелким серым или заголовок скрыт, а
подпись сделана ячейкой (стиль превью).

**7.4. Условное форматирование.** Правила в порядке: «пусто» → «сегодня» →
шкала цвета (тепловая карта); проверка после вставки строк (УФ должен
покрывать открытый диапазон, напр. `C4:I9`, а не отдельные ячейки).

**7.5. Защита.** Ячейки ввода (чекбоксы, даты, суммы, названия в Настройках) —
единственное редактируемое. Всё остальное + `_calc` целиком — под защитой с
предупреждением. Проверить: попытка стереть KPI выдаёт «Вы пытаетесь изменить
защищённую ячейку».

**7.6. Мобильная читаемость.** Открыть с телефона в приложении Google
Таблицы: KPI-плитки видны без зума на первом экране; закреплена строка
заголовка на листах ввода (Вид → Закрепить); чекбоксы попадают под палец
(высота строк ≥ 28 пт); дашборд листается только вниз, не вбок.

**7.7. Инструкция новичку.** На листе Настройки сверху блок «Старт за
60 секунд», максимум 5 строк: 1) Файл → Создать копию; 2) впиши свои
привычки и цены; 3) ставь галочки на листе Привычки; 4) смотри дашборд;
5) не редактируй серые ячейки — там формулы. Тон — дружелюбный, без сленга,
тире — настоящие «—».
