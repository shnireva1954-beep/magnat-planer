# ТЗ: реальные Google-таблицы «Магнат 5 в 1»

**Цель:** каждый реальный лист визуально и функционально соответствует превью на
magnat-planer.ru. Всё в этом ТЗ реализуемо штатными средствами Google Sheets —
превью проектировались с этим ограничением. Критерий готовности: скриншот
реального дашборда не стыдно поставить на лендинг вместо макета.

**Продавать до соответствия таблиц превью нельзя** — это прямой путь к
возвратам («на сайте не то, что мне прислали»).

---

## 0. Общая структура каждого файла

Каждый планер — отдельный файл Google Sheets с листами:

| Лист | Назначение |
|---|---|
| **📊 Дашборд** | Витрина: KPI-плитки, диаграммы, инсайт. Пользователь сюда смотрит |
| **Данные** (Привычки/Задачи/…) | Рабочий лист: сюда ставят галочки/вводят записи |
| **Настройки** | Список привычек и цены в монетах, цели, пороги уровней |
| *(скрытый)* **_calc** | Промежуточные формулы, чтобы Дашборд был чистым |

Оформление в стиле превью: белый фон, шапки `#eef2f8`, сетка `#e0e6ef`,
акценты: изумруд `#12a565`, золото `#c98a12`, красный `#e0556a`,
фиолетовый `#7a52d0`, синий `#2f8fd8`. Защита листов: всё, кроме ячеек ввода,
защитить (Данные → Настроить защищённые листы), иначе пользователи стирают
формулы — главная причина обращений в поддержку.

## 1. Игровая механика (одинаково во всех планерах, сверено с приложением)

- **Монеты за день** = `СУММПРОИЗВ(галочки_дня; цены_привычек)`
  (цены задаются в Настройках, по умолчанию 10–30).
- **День засчитан** в серию, если выполнено ≥ 3 привычек:
  `=ЕСЛИ(СЧЁТЕСЛИ(диапазон_дня;ИСТИНА)>=3;1;0)`.
- **Серия** — количество зачтённых дней подряд до сегодня (в _calc:
  бегущий счётчик `=ЕСЛИ(день_зачтён=1;пред_серия+1;0)`).
- **Множитель** = `МИН(2; 1 + 0,1 * серия_до_вчера)`.
- **Монеты с множителем** = `ОКРУГЛ(монеты_дня * множитель)`.
- **Уровень** по суммарным монетам, пороги как в приложении:
  `=ПРОСМОТР(всего_монет; {0;300;1500;5000;15000}; {"🐣 Новичок";"🚀 Стартапер";"💼 Предприниматель";"🏢 Директор";"👑 Магнат"})`.
- Прогресс до следующего уровня — SPARKLINE-бар:
  `=SPARKLINE(монет_в_уровне; {"charttype";"bar"; "max";размер_уровня; "color1";"#12a565"})`.

## 2. Планер привычек — «Штаб дисциплины»

**Данные:** строки — привычки (эмодзи+название из Настроек), столбцы — дни
месяца, ячейки — чекбоксы (Вставка → Флажок). Колонка `%` =
`СЧЁТЕСЛИ(строка;ИСТИНА)/прошедшие_дни`.

**Дашборд:**
- KPI-плитки (крупные ячейки с заливкой и цветной левой границей):
  Серия 🔥 · Множитель · Монеты сегодня · Уровень (формулы из п.1).
- **Кривая прогресса** — диаграмма «Область» по % выполнения каждого дня.
- **Пончик «Месяц»** — круговая с отверстием: выполнено/не выполнено.
- Таблица недели: срез Данных за текущую неделю (`ФИЛЬТР` по датам).
- Прогресс уровня — SPARKLINE-бар (п.1).
- **Инсайт** (одна ячейка, стиль «💡 на светло-жёлтом»):
  - сильный день: `="Твой сильный день - "&ИНДЕКС(дни;ПОИСКПОЗ(МАКС(проценты_по_дням);проценты_по_дням;0))&" ("&ТЕКСТ(МАКС(...);"0%")&")"`
  - хвост про множитель: `"Ещё 1 день серии - множитель "&ТЕКСТ(МИН(2;1,1+0,1*серия);"0,0")`.

## 3. Планер задач — «Центр задач»

**Данные:** колонки Задача · Срок (дата) · Приоритет (раскрывающийся список
Высокий/Средний/Низкий) · ✓ (чекбокс). Колонка «Осталось» =
`=ЕСЛИ(✓;"✓";срок-СЕГОДНЯ())`.

**Условное форматирование:** просрочка (`осталось<0 И НЕ ✓`) — красный текст;
выполненные — серые + зачёркнутые (формат по ✓=ИСТИНА); приоритеты — цветные
пилюли (заливка по значению).

**Дашборд:** KPI: Открыто · Горит (просрочено) · Закрыто за неделю · % в срок.
**Столбчатая диаграмма** «закрыто по дням недели» (`СЧЁТЕСЛИМН` по дате
закрытия — добавить скрытую колонку «дата выполнения», проставляется вручную
или скриптом). Инсайт: пик продуктивности + самая старая просрочка
(`ИНДЕКС/МИН` по просроченным).

## 4. Фитнес-планер — «Панель формы»

**Данные:** дневник — Дата · Вес · Ккал · Вода · Сон · Тренировка (чекбокс) ·
Группа мышц (список) · Настроение. Настройки: цель веса, норма ккал/воды.

**Дашборд:**
- KPI: Вес (последний: `=ПРОСМОТР(2;1/(веса<>"");веса)`) · Δ за месяц ·
  Тренировки за 30 дн (`СЧЁТЕСЛИМН`) · Сон средний.
- **Путь к цели**: старт → текущий → цель, прогресс-бар SPARKLINE
  `=SPARKLINE((старт-тек)/(старт-цель);{"charttype";"bar";"max";1})`,
  подпись «осталось X кг · N% пути».
- **График веса** — «Область»: серия веса + константная серия «цель»
  (колонка, залитая значением цели) → пунктирная линия цели на графике
  (стиль линии серии: пунктир, золото).
- **Радар «Баланс мышц»** — диаграмма «Лепестковая» по `СЧЁТЕСЛИ` подходов
  на каждую группу за 30 дней. Radar есть в Sheets: тип «Лепестковая».
- Инсайт: `МИН`-группа → «Плечи отстают - добавь подход».

## 5. Планер недели — «Пульт недели»

**Данные:** 7 блоков дня: 3 главных дела (чекбоксы) · энергия (1–5) ·
настроение (список эмодзи). Отдельно: 3 фокус-цели недели (чекбоксы).

**Дашборд:** KPI: Выполнено X/Y · Энергия средняя · Фокус-цели.
Мини-карточки дней (как в превью): дела, шкала энергии
`=SPARKLINE(энергия;{"charttype";"bar";"max";5})`, кольцо дня — мини-пончик
или `ТЕКСТ(процент;"0%")` с цветом. Инсайт: день с минимальной энергией →
«Ставь лёгкие задачи на …».

## 6. Финансовый планер — «Капитал»

**Данные:** журнал операций — Дата · Тип (доход/расход) · Категория (список
с эмодзи) · Сумма. Лист «Планы»: категория → план на месяц. Лист «Долги».

**Дашборд:**
- KPI: Доход · Расход · Сбережения (=Д-Р, + `"("&ТЕКСТ(С/Д;"0%")&" дохода)"`) ·
  Подушка (сбережения_накоплено / средний_расход, «X мес»).
- **Пончик расходов** по категориям (`СУММЕСЛИ` по категориям → круговая
  с отверстием).
- **План vs факт**: таблица категорий + SPARKLINE-бары факта, факт>план —
  условное форматирование красным.
- **Копилка по месяцам** — столбчатая по `СУММЕСЛИМН` (доход-расход) за
  последние 6 месяцев.
- Инсайт: `="Сбережения "&ТЕКСТ(С/Д;"0%")&" дохода"&ЕСЛИ(есть_перерасход;" · "&категория&" превысила план на "&ТЕКСТ(дельта;"# ##0 ₽");"")`.

## 7. Чек-лист сдачи каждого планера

- [ ] Дашборд визуально совпадает с превью на лендинге (палитра, состав блоков)
- [ ] Все формулы работают при добавлении/удалении строк пользователем
- [ ] Ячейки ввода — единственные незащищённые; остальное под защитой
- [ ] Инструкция «Как настроить под себя» на листе Настройки (5 строк максимум)
- [ ] Файл открыт «для просмотра всем у кого есть ссылка» → покупатель делает
      «Файл → Создать копию»
- [ ] Проверено с телефона в приложении Google Таблицы (читается ли дашборд)

После готовности: заменить/дополнить HTML-превью на лендинге реальными
скриншотами и дать ссылку «Открыть демо-таблицу» (view-only копия с
демо-данными) — это сильнейший аргумент доверия.
