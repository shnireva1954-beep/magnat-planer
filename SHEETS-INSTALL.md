# Как собрать 5 таблиц «Магнат» за 10 минут

Файл `SHEETS-BUILDER.gs` — готовый скрипт, который сам создаёт все пять
таблиц в твоём Google-аккаунте: листы, формулы, чекбоксы, диаграммы,
тепловую карту, защиту и демо-данные. Руками ничего строить не нужно.

## Шаг 1. Запусти скрипт (5 минут)

1. Открой [script.google.com](https://script.google.com) под своим
   Google-аккаунтом → кнопка **«Создать проект»**.
2. В редакторе удали заготовку (`function myFunction() {…}`).
3. Открой файл `SHEETS-BUILDER.gs` из репозитория, скопируй **весь** текст
   и вставь в редактор. Нажми 💾 (или Ctrl+S).
4. Сверху в выпадающем списке функций выбери **buildAll** и нажми
   **«Выполнить»**.
5. Google спросит разрешения: «Разрешить» → если появится «Приложение не
   проверено» — жми **«Дополнительно» → «Перейти к проекту (небезопасно)»**.
   Это твой собственный скрипт в твоём аккаунте, он никуда ничего не
   отправляет — весь код открыт в этом файле.
6. Подожди 1–2 минуты. Внизу в «Журнале выполнения» появятся ссылки.

Готово: на Google Диске появилась папка **«Магнат · Планеры»** с пятью
таблицами. В каждой — демо-данные за последние недели (относительно
сегодняшней даты), чтобы дашборды сразу были живыми.

## Шаг 2. Финальные штрихи руками (5 минут)

Две вещи Apps Script не умеет — доделай в паре кликов:

1. **Число в центре пончиков.** Открой диаграмму-пончик → рядом вставь
   диаграмму «Сводная карточка» (Вставка → Диаграмма → тип «Сводная
   карточка»), укажи ячейку с процентом/суммой (в скрытом листе `_calc`),
   фон карточки поставь «Нет» и перетащи её поверх отверстия пончика.
   Хочешь проще — пропусти: пончики работают и без этого.
2. **Пунктир линии цели** (таблица «Тело»): если линия цели на графике веса
   сплошная — открой редактор диаграммы → Серии → «Цель» → «Тип штриха
   линии» → пунктир.

## Шаг 3. Проверь по чек-листу (из SHEETS-SPEC.md)

- [ ] Открой каждый дашборд — нет ли ячеек с #ОШИБКА/#REF/#ДЕЛ0.
- [ ] Поставь пару галочек на листе «Привычки» — монеты и серия меняются.
- [ ] Сделай «Файл → Создать копию», сотри демо-данные (только данные,
      не формулы!) — дашборд показывает нули и подсказки, не ошибки.
- [ ] Открой с телефона в приложении Google Таблицы — KPI читаются.
- [ ] Попробуй стереть ячейку дашборда — должно выскочить предупреждение.

## Шаг 4. Подготовь к продаже

1. В каждой таблице: **Файл → Поделиться → Доступ по ссылке →
   «Читатель»**. Покупатель делает «Файл → Создать копию» — и таблица его.
2. Эти 5 ссылок вставь в письмо после оплаты в ЛК Робокассы (плюс ссылку
   на приложение https://magnat-planer.ru/app.html).
3. Сделай скриншоты дашбордов (с демо-данными) — заменим ими превью на
   лендинге, и добавим кнопку «Открыть демо-таблицу».

## Если что-то пошло не так

- Скрипт упал с ошибкой — скопируй текст ошибки из журнала и пришли в чат:
  поправлю за один заход.
- Какая-то диаграмма не создалась — скрипт про это напишет в журнале
  (он не падает из-за одной диаграммы), её можно добавить руками по
  описанию из SHEETS-SPEC.md.
- Запускать buildAll повторно безопасно: создаётся НОВЫЙ комплект таблиц
  (старые остаются, лишние просто удали).
